10 Most asked Soft Skills questions

  • 1.WHAT IS STLC
    STLC(SOFTWARE TESTING LIFE CYCLE)
    Sequence of different activities performed during the software testing process
    Testing process which has specific steps to be executed in a definite sequence to ensure that the quality goals have been met.
    In the STLC process, each activity is carried out in a planned and systematic way.
    STLC phases:
    Requirement, Planning,Analysis,Design,Implementation,Execution,Conclusion,Closure


  • 2.WHAT IS SDLC?
    SDLC (SOFTWARE DEVELOPMENT LIFE CYCLE)
    Sequence of different activities performed during the software development process
    Systematic process for building software that ensures the quality and correctness of the software built.
    SDLC process aims to produce high-quality software that meets customer expectations.
    SDLC consists of a detailed plan which explains how to plan, build and maintain specific software.
    SDLC phases:
    Requirement , Planning, Design, Coding, Testing.


  • 3.WHY DO SOFTWARE APPLICATIONS HAVE BUGS?\
    There are various reasons why a system might have bugs, such as bad design, coding errors, miscommunication, continuous change in requirements, rushing to meet deadlines and also the complexity of the application.


  • 4.CAN YOU TELL THE DIFFERENCE BETWEEN VERIFICATION AND VALIDATION
    VERIFICATION
    When you are analyzing test cases to ensure we cover all scenarios and we are creating the right product and NO coding is involved
    Verification is the static testing.
    Verification means: Are we building the product right?

    VALIDATION
    Is when we are executing test cases and writing code to verify the system.
    Validation is dynamic testing.
    Validation means: Are we building the right product?


  • 5.WHAT IS THE DIFFERENCE BETWEEN STATIC AND DYNAMIC TESTING?
    STATIC TESTING(done in verification stage)
    Is a White box Testing technique where the developers verify or test their code with the help of a checklist to find errors in it, this type of testing is done without running the developed application or program.
    DYNAMIC TESTING(done in validation stage)
    Is done by executing the actual application with valid inputs to check the expected output.
    Examples: Integration Testing, System testing, Regression and Acceptance Testing.


  • 6.WHAT IS THE DIFFERENCE BETWEEN BLACK BOX AND WHITE BOX TESTING?
    BLACK BOX TESTING
    Is a software testing method in which the internal structure of the item being tested is NOT known to the tester
    It is a technique that testers do without knowing the internal structure of the code.
    Mainly applicable to SMOKE & REGRESSION TESTING
    We only do black box testing
    WHITE BOX TESTING
    Is a software testing method in which the internal structure of the item being tested is known by the tester
    Mainly applicable to lower levels of testing: UNIT TESTING & INTEGRATION TESTING
    If you are able to see the code, it is White Box Testing , any testing that is done by developer is White Box Testing, for us to be able to do WHITE BOX TESTING we need to know the internal structure of the code, which means we need to access the code, but as a tester we don’t have access to developer code

  • 8.IN WHICH PHASE SHOULD TESTING BEGIN?

    The important entry criteria for testing are:
    Once a test plan, test cases are written and approved
    Once the environment is ready
    Once the build is given to QA team to test
    Test data is ready


  • 9.What is AD-Hoc testing?
    Informal Testing which is performed by testers
    Application is tested randomly with random inputs
    When you do it you don’t need documentation, test plan, test case
    Ad-hoc testing is usually done to discover the issues or defects which cannot be found by following the formal process.
