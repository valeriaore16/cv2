Tell me about your framework

  • In my framework, I was developing my testing code using Java programming language and Selenium WebDriver.
    I used Maven to manage and centralize my dependencies and Cucumber and Selenium as dependencies in my framework.
    Since it is Behavior Driven Development(BDD) I created one of my feature files where I use the Gherkin language to create my scenarios and scenario outline.
    My framework was structured based on the POM(Page Object Model) that every page element goes to one class and it makes maintenance easy to maintain the Web Elements and to keep my code organized and clean.
    I also have a step definition package where I implemented every step of feature file scenarios.
    I have created other separate folders for my runner classes for regression and smoke tests and for other runners.
    At the same time, I store all my utilities for JDBC, ExcelUtils, Configuration Util, Reusable Selenium Utils and Driver classes under the Utils package.
    And I achieve continuous integration and schedule my Regression suite and Smoke test using Jenkins and it is generating the cucumber html report.
    I also push my code to the remote repository using Git with GitHub.
